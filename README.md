# Лабораторная работа №12 — Работа с транспортной компанией

## Цель работы
Создать систему для работы с клиентами и транспортными средствами компании.  
Реализовать классы для клиентов и транспорта, а также класс для транспортной компании с функционалом распределения грузов.

---

## Требования к практической части

- Программы должны **запрашивать ввод данных от пользователя** и выводить результаты на экран с пояснениями.
- Каждое задание должно находиться в отдельной папке `task_Y` в репозитории `ipo-lr-12`, где `Y` — номер задания.
- Репозиторий должен содержать:
  - пакет **transport** с файлами классов,
  - файл **main.py** для запуска программы с меню.
- Каждый класс должен находиться в отдельном модуле.
- Для всех методов должна присутствовать **валидация данных**.

---

## Задание 1 — Класс Client

Создать класс **Client** для клиентов компании.

**Атрибуты:**
- `name` — имя клиента
- `cargo_weight` — вес груза
- `is_vip` — флаг VIP-статуса (по умолчанию False)

**Требования:**
- Валидация данных (тип и положительность груза)
- Строковое представление клиента через `__str__`

---

## Задание 2 — Базовый класс Vehicle

Создать базовый класс **Vehicle** для транспортного средства.

**Атрибуты:**
- `vehicle_id` — уникальный идентификатор (строка, генерируется автоматически)
- `capacity` — грузоподъемность (в тоннах)
- `current_load` — текущая загрузка (по умолчанию 0)
- `clients_list` — список клиентов, чьи грузы загружены

**Методы:**
- `load_cargo(client)` — загружает груз клиента с проверкой на вместимость и тип данных
- `__str__` — возвращает строковое представление транспорта

---

## Задание 3 — Индивидуальные задания

**1. Класс Truck (грузовик)**  
- Наследует Vehicle  
- Дополнительный атрибут: `color` (цвет грузовика)

**2. Класс Train (поезд)**  
- Наследует Vehicle  
- Дополнительный атрибут: `number_of_cars` (количество вагонов)

**3. Класс TransportCompany**  
**Атрибуты:**
- `name` — название компании  
- `vehicles` — список транспортных средств  
- `clients` — список клиентов  

**Методы:**
- `add_vehicle(vehicle)` — добавить транспортное средство с валидацией  
- `list_vehicles()` — вернуть список всех транспортных средств  
- `add_client(client)` — добавить клиента  
- `optimize_cargo_distribution()` — распределяет грузы по транспортным средствам:  
  - сначала VIP клиенты  
  - минимальное количество используемого транспорта  

---

## Задание 4 — Файл main.py

Создать **файл main.py** с меню для работы с системой:

**Функционал меню:**
1. Добавление клиента  
2. Добавление транспортного средства (грузовик/поезд)  
3. Показ всех транспортных средств  
4. Распределение грузов клиентов по транспорту  
5. Выход из программы  

**Примечания:**
- Использовать классы из пакета `transport`  
- Все действия должны сопровождаться пояснением для пользователя  
- Меню должно быть в цикле, пока пользователь не выберет выход

---

## Общие требования

- Валидация всех входных данных  
- Каждый класс и метод снабжен комментариями  
- Репозиторий содержит:
  - папки `task_1`, `task_2`, `task_3`, `task_4`  
  - пакет `transport` с модулями классов  
  - файл `main.py` с меню
